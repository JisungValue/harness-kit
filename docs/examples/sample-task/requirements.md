# Requirements

## 기능 요구사항

- 프로젝트 로컬 entrypoint가 공통 Harness Core Guide를 참조해야 한다.
- 프로젝트 전용 문서 위치가 함께 연결되어야 한다.
- 새 작업자가 guide만 보고 공통 규칙과 프로젝트 전용 규칙의 경계를 이해할 수 있어야 한다.

## 비기능 요구사항 또는 품질 요구사항

- 문서 구조는 단순해야 한다.
- 공통 규칙과 프로젝트 규칙이 서로 중복되지 않아야 한다.

## 입력/출력

- 입력: Harness Kit
- 출력: 프로젝트 로컬 참조 guide와 project overlay 문서 세트

## 제약사항

- 공통 Kit를 프로젝트 전용 규칙으로 오염시키지 않는다.
- 프로젝트 전용 세부 규칙은 overlay 문서에만 둔다.

## 예외 상황

- 특정 프로젝트가 kit 구조를 그대로 따르기 어려우면 로컬 guide에 예외 사유를 남긴다.

## 성공 기준

- 공통 문서와 프로젝트 전용 문서 연결이 문서화된다.
- 새 task 시작 시 어디서 읽고 어디에 기록할지 혼동이 없다.
