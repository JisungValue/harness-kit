# Audit Policy

## 목적

이 문서는 모든 Phase 감사에서 공통으로 적용되는 운영 규칙을 정의한다.

## 언제 다시 읽는가

- 각 Phase에서 감사를 수행하기 직전에 다시 읽는다.
- 감사 결과 형식, 피드백 분류, 미확인 입력물 표기 방식이 애매할 때 다시 참조한다.

## 공통 감사 입력물 확인 규칙

감사자는 Phase 전용 입력물 외에도 아래 항목을 함께 확인한다.

- 해당 task의 `issue.md`
- 해당 task의 현재 Phase 산출물
- 이전 Phase에서 현재 Phase 판단에 영향을 주는 산출물
- 관련 공통 정책 문서
- 관련 프로젝트 overlay 문서

입력물 중 일부를 읽지 못했거나 존재하지 않으면 그 사실을 감사 결과에 명시한다.

## 공통 감사 운영 규칙

- 감사는 해당 Phase 산출물과 관련 입력물을 기준으로 수행한다.
- 감사가 승인 불가이면 피드백을 반영한 뒤 같은 Phase에서 재감사한다.
- 감사자는 범위 위반, 책임 혼합, 테스트 부족, 검증 부족, 문서 반영 누락을 우선적으로 본다.
- 감사자는 프로젝트 전용 규칙이 필요한 판단이면 overlay 문서를 함께 참조한다.

## 공통 감사 피드백 분류 규칙

피드백은 가능하면 아래 분류 중 하나 이상으로 구분한다.

- 누락
- 모호함
- 구조 위반
- 범위 확장
- 테스트 부족
- 검증 부족
- 문서화 부족

## 감사 결과 형식

- 판정: 승인 가능 / 승인 불가
- 피드백: 보완이 필요한 항목 목록
- 피드백 분류: 위 분류 기준 사용
- 미확인 입력물: 없으면 없음으로 표기
